# Windows 桌面小组件 — 轻量级方案

> 常驻桌面的一小块信息面板：时间、农历、天气、今日待办。  
> **原则：能省则省 — 小体积、低内存、少依赖、单窗口、本地优先。**

---

## 0. 轻量级定义（本项目的硬指标）

| 指标 | 目标 | 不做 |
|------|------|------|
| 安装包 | **< 15 MB**（框架依赖型）或 **< 40 MB**（单文件自包含） | 不搞 Electron、不搞多模块解决方案 |
| 常驻内存 | **< 40 MB** 空闲 | 不常驻 WebView、不后台多进程 |
| 项目结构 | **单项目、单 exe** | 不拆 Core/Data/Providers 多工程 |
| 依赖 | **0～2 个 NuGet** | 不用 EF、不用 MVVM 框架、不用 CommunityToolkit（初期） |
| 存储 | **两个 JSON 文件**（data + settings） | 不用 SQLite；扩展功能仍写进同一 data.json |
| 网络 | **仅天气** 可选联网 | 无账号、无云同步、无 OAuth |
| UI | **一种紧凑布局** + 模块开关 | 不做拖拽排版、不做皮肤市场 |
| 设置 | **托盘右键菜单** 为主 | 不做独立大型设置页 |

## 1. 产品定位

### 1.1 一句话

**桌面上一条「今日信息条」** — 抬头看时间农历天气，顺手勾待办，不占地方、不抢资源。

### 1.2 轻量版 vs 完整版

原先设想里的「驾驶舱」偏全功能。轻量版做减法：

| 完整版（暂缓） | 轻量版（现在做） |
|----------------|------------------|
| 月历 + 节假日调休标注 | **今日日期行** + 迷你周历（7 格） |
| Todo + 独立计划表 + 时间轴 | **合并为「今日清单」**，可选填时间 |
| 多时区、秒针动画 | **单时钟**，12/24 可切 |
| 7 日天气预报 | **今明两天** 温度 + 图标 |
| 宜忌、AQI、展开详情 | **全部砍掉** |
| Outlook / Google 同步 | **不做** |
| 4 种布局预设 | **1 种布局**，可调宽高 |

### 1.3 核心价值（轻量语境下）

- **几乎无感**：小块透明卡片，可置顶、可穿透
- **开机即用**：无配置也能看时间农历；天气填城市或自动定位
- **数据在本地**：一个 `data.json`，可复制备份
- **不装一堆运行时**：优先依赖系统已装的 .NET，而非自带整套 SDK

## 2. 功能范围（MVP 锁定）

### 2.1 唯一布局 — 紧凑竖条

```
┌─────────────────────────────┐
│  14:32        周四 6月11日    │  ← 时间 + 公历
│  农历五月初六 · 芒种          │  ← 农历 + 节气（一行）
│  ☀ 28°  晴  18°~31°         │  ← 天气（可关）
├─────────────────────────────┤
│  ▢ 买牛奶                    │  ← 今日待办（最多显示 5 条）
│  ▢ 14:00 周会               │     有时间则前缀显示
│  ▢ 写周报                    │
│  + 添加…                     │  ← 点击弹出输入，或托盘快捷键
├─────────────────────────────┤
│  一 二 三 四 五 六 日         │  ← 迷你周历（今天高亮）
│     ●                       │
└─────────────────────────────┘
     约 280 × 360 px
```

模块均可单独关闭：只要时钟、只要待办、只要天气，都行。

### 2.2 模块清单

#### 时间（必开，不可关）

- 时:分（默认不显示秒，**省定时器刷新**）
- 公历：月日、星期
- 设置：12 / 24 小时制

#### 万年历（轻量版）

- 农历月日（如「五月初六」）
- 当日节气 / 传统节日（有则显示，**一行挤下**）
- 迷你周历：当前周 7 天，今天圆点高亮
- **不做**：月视图翻页、宜忌、调休、点击日期弹层

#### 天气（可选模块）

- 城市、天气图标、当前温、今日高低温
- 数据源：**Open-Meteo**（免费、无需 API Key）
- 刷新：**60 分钟**一次；失败用上次缓存
- 定位：IP 粗定位或手动填城市名（**不做** GPS 持续定位）

#### 今日清单（Todo + 计划表合并）

- 字段：`标题` + 可选 `时间(HH:mm)` + `完成`
- 只显示**今天**的条目；无日期或日期=今天的都进今日
- 最多展示 5 条，超出托盘菜单「查看全部」打开小窗口（第二阶段）
- 本地 JSON 持久化，**无优先级、无标签、无子任务**

### 2.3 明确不做的功能（防止膨胀）

- 云同步、账号登录
- Outlook / Google / 微软 To Do 集成
- 重复日程规则引擎（每天/每周/每月）
- 拖拽排版、插件市场、皮肤商店
- 宜忌、独立天气详情页、7 日预报大面板
- 多语言（先只做简体中文）
- 内嵌浏览器、RSS 阅读器、股票行情

## 2.4 扩展功能设想（轻量可插拔）

> **原则**：以下功能全部 **默认关闭**，在托盘里按需开启；实现上优先「本地计算 / 同一 JSON / 同一天气 API」，不新增重型依赖。  
> 每个模块 UI 占用 **≤ 2 行**，可折叠，避免面板变成长条瀑布。

### 功能地图

```
                    ┌─ 感知层（看）
                    │   时间 / 农历 / 天气 / 月相 / 日出日落
                    │
  DeskLite 核心 ────┼─ 行动层（做）
                    │   今日清单 / 倒数日 / 习惯打卡 / 番茄钟
                    │
                    └─ 陪伴层（省）
                        每日一句 / 护眼提醒 / 进度条 / 快捷启动
```

### A 类 — 强烈推荐（性价比高，仍轻量）

| 模块 | 展示示例 | 实现成本 | 说明 |
|------|----------|----------|------|
| **倒数日** | `距离春节还有 214 天` | 低 | 用户自建事件（考试、发薪、纪念日、项目上线）；内置常见节日表，本地算 |
| **今年/本周进度** | `2026 已过 45.2% ▓▓▓░░` | 低 | 纯日期计算；可切换「本周已过 %」「距周末 2 天」 |
| **待办到时提醒** | 托盘 Toast：`14:00 周会` | 低 | 仅针对**今天且写了时间**的待办；到点弹一次，不搞复杂重复规则 |
| **日出 / 日落** | `日出 04:51 · 日落 19:32` | 低 | Open-Meteo 同一请求顺带返回，不增加 API |
| **月相** | `🌔 盈凸月` | 低 | 本地天文算法，一行图标+文字 |
| **每日一句** | `行胜于言。` | 低 | 内置 100～300 条本地语录，按日期种子随机，**完全离线** |
| **速记便签** | 一行可编辑文字 | 低 | `scratch` 字段写入 JSON，适合临时手机号、会议号 |
| **置顶待办** | ⭐ 标记 1 条 | 低 | 今日清单内最多置顶 1 条，始终显示在最上 |

### B 类 — 很实用（略增交互，仍可控）

| 模块 | 展示示例 | 实现成本 | 说明 |
|------|----------|----------|------|
| **习惯打卡** | `喝水 ●○○  早睡 ●●○` | 中 | 用户定义 ≤5 个习惯，每日点一下；零点自动重置 |
| **番茄钟** | `专注 18:32 ▶` | 中 | 25/5 默认，托盘显示倒计时；**不另开窗口**，折叠区展开才操作 |
| **护眼提醒** | 每 20 分钟托盘提示 | 低 | 20-20-20 规则；仅 Toast + 可选弱提示音，**不抢焦点** |
| **快捷启动** | 4 个图标/文字按钮 | 中 | 用户拖入 exe/文件夹/网址，一键打开；数据存 JSON 路径 |
| **第二时区** | `东京 15:32` | 低 | 仅 **1 个**副时钟，与主时钟共用分钟 tick |
| **节假日标注** | 周历上 `休` `班` 小字 | 中 | 国务院节假日 **年度 JSON 包**（每年更新一次内置文件），离线 |
| **查看全部待办** | 小弹窗列表 | 中 | 待办 >5 条时托盘打开；简单列表，可搜标题，无标签体系 |
| **数据备份** | 导出/导入 JSON | 低 | 托盘一键导出 `desk-lite-backup-日期.json` |

### C 类 — 锦上添花（按需开启）

| 模块 | 展示示例 | 实现成本 | 说明 |
|------|----------|----------|------|
| **生日/纪念日** | `3 天后 妈妈生日` | 中 | 与倒数日共用数据，按年重复；提前 3 天开始在面板提示 |
| **今明天气对比** | `明天 小雨 22°~26°` | 低 | 天气 API 多取一天，一行展示 |
| **UV / 降水概率** | `UV 7 · 降水 30%` | 低 | 天气 API 扩展字段，一行 |
| **电量/充电** | `🔋 76%` | 低 | 笔记本 `SystemInformation.PowerStatus`；台式机自动隐藏 |
| **下一待办倒计时** | `1h20m 后 周会` | 低 | 取最近一条未完成的定时待办 |
| **透明度/字号** | 托盘滑块 | 低 | 无设置页，滑块调节面板透明度与字体 |
| **贴边收纳** | 鼠标移开自动缩成细条 | 中 | 仅露时间一行，悬停展开；适合靠边放置 |
| **全局快捷键** | `Ctrl+Shift+N` 快速记待办 | 低 | 弹出极简输入框，回车即保存 |

### D 类 — 暂不纳入（偏重或与定位不符）

| 功能 | 不做的原因 |
|------|------------|
| CPU/内存/网速实时监控 | 需高频采样，占 CPU，违背「无感」 |
| 音乐「正在播放」 | 需对接 Spotify/WMI，脆弱且依赖多 |
| 剪贴板历史 | 隐私敏感，易误触泄露 |
| 星座运势 / 宜忌 | 争议内容，与工具定位不符 |
| 股票 / 加密货币 | 需 API Key + 高频刷新 |
| 内嵌 RSS / 新闻 | 易变 Web 内容，倾向做浏览器的事 |
| 多窗口 / 多显示器多实例 | 复杂度陡增，后期再评估 |

## 4. 技术方案（轻量优先）

### 4.1 推荐技术栈

| 层级 | 选型 | 轻量理由 |
|------|------|----------|
| UI | **WPF** (.NET 8) | 比 WinUI 3 依赖小；透明窗口成熟；资料多 |
| 模式 | **Code-behind + 少量 Service** | 不引入 MVVM 框架；文件少 |
| 存储 | **`%AppData%/DeskLite/data.json`** | 无需数据库 |
| 配置 | **同目录 `settings.json`** | 与数据分离 |
| 农历 | **内嵌一小段算法** 或单个 `.cs` 文件 | 不引大型日历库 |
| 天气 | `HttpClient` + JSON 解析 | 一个类搞定 |
| 托盘 | `System.Windows.Forms.NotifyIcon` | WPF 里常用，就一个图标 |
| 发布 | **框架依赖** `win-x64` | 体积最小（用户需 .NET 8） |

**为何不用 WinUI 3（轻量语境）**

- Windows App SDK 运行时体积大，对小工具过重
- 若以后要做 Mica 精致效果，再评估迁移；MVP 用 WPF 半透明 + 圆角 `Border` 足够

**为何不用 Electron / Tauri**

- 内存通常 100MB+，与「轻量」目标相反

### 4.2 项目结构（单项目）

```
DeskLite/
├── App.xaml
├── MainWindow.xaml          # 唯一主窗口
├── Services/
│   ├── ClockService.cs      # 每分钟 tick 一次（无秒）
│   ├── LunarService.cs      # 农历/节气
│   ├── WeatherService.cs    # Open-Meteo + 缓存 + 日出/明日
│   ├── TodoStore.cs         # 读写 data.json
│   ├── CountdownService.cs  # 倒数日 / 生日
│   ├── HabitService.cs      # 习惯打卡（可选）
│   └── PomodoroService.cs   # 番茄钟（可选）
├── Models/
│   ├── TodoItem.cs
│   ├── Countdown.cs
│   └── AppSettings.cs
├── TrayIcon.cs
└── DeskLite.csproj          # 就一个项目
```

MVP 约 **15 个文件**；扩展模块逐步加，**控制在 25 个文件以内**。

### 4.4 性能策略

- 时钟：**每分钟**更新 UI 一次（不显示秒）
- 天气：启动拉一次，之后每 60 分钟；窗口激活时不额外刷新
- 农历：日期变更时算一次，缓存当天结果
- 不用 `DispatcherTimer` 堆多个，**一个定时器**管时钟

## 5. 体积与内存估算

| 发布方式 | 大约体积 | 说明 |
|----------|----------|------|
| 框架依赖 `dotnet publish -c Release` | **~2–5 MB** exe + 依赖 DLL | 需用户安装 [.NET 8 桌面运行时](https://dotnet.microsoft.com/download) |
| 自包含单文件 `PublishSingleFile` | **~35–45 MB** | 免装运行时，适合「解压即用」 |

| 状态 | 内存目标 |
|------|----------|
| 空闲展示 | **25–40 MB** |
| 刚启动 | **< 2 秒** 出界面 |
